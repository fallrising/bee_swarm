# Bee Swarm 文件檢查計劃

## 🎯 檢查目標

基於 `CONTEXT.md` 核心思想，系統性檢查現存文件是否：
1. **符合核心定位**：專注協作模式設計而非生產實現
2. **遵循設計哲學**：體現簡化、透明、異步、可驗證原則  
3. **滿足技術約束**：在 AI 工具、基礎設施、GitHub 約束範圍內
4. **體現核心價值**：有助於實現概念先行、數據驅動等價值
5. **支持混合架構**：為容器+Actions混合模式留有空間

## 📋 檢查清單模板

### 文件檢查標準
對每個文件評估以下維度：

```markdown
## 文件：{文件路徑}

### ✅ 符合性檢查
- [ ] **核心定位符合**：是否專注協作模式設計？
- [ ] **設計哲學遵循**：是否體現四大設計原則？
- [ ] **技術約束滿足**：是否在約束範圍內？
- [ ] **核心價值實現**：是否體現四大核心價值？
- [ ] **混合架構支持**：是否考慮混合模式？

### 🔍 具體問題
- **不符合項**：列出具體不符合的地方
- **改進建議**：提出具體的改進方案
- **優先級**：高/中/低

### 📊 評分（1-5分）
- 核心定位：[ ]/5
- 設計哲學：[ ]/5  
- 技術約束：[ ]/5
- 核心價值：[ ]/5
- 混合架構：[ ]/5
- **總分：[ ]/25**
```

## 🛠️ 檢查工具

### 自動化檢查腳本
```bash
#!/bin/bash
# check_file_compliance.sh

FILE_PATH=$1
CONTEXT_FILE="CONTEXT.md"

echo "檢查文件：$FILE_PATH"
echo "參考標準：$CONTEXT_FILE"

# 檢查關鍵詞符合性
check_keywords() {
    echo "=== 關鍵詞檢查 ==="
    grep -i "簡化\|transparency\|異步\|async" "$FILE_PATH" || echo "缺少設計哲學關鍵詞"
    grep -i "github\|container\|action" "$FILE_PATH" || echo "缺少技術架構關鍵詞"
    grep -i "模擬\|simpy\|驗證" "$FILE_PATH" || echo "缺少驗證相關關鍵詞"
}

# 檢查結構完整性
check_structure() {
    echo "=== 結構檢查 ==="
    # 檢查是否有適當的章節結構
}

check_keywords
check_structure
```
